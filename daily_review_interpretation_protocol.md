# A 股主线日报复盘解读协议

本协议用于用户要求助手解读每日 HTML/Markdown 日报时使用。

## 核心原则

```text
日报生成阶段不调用大模型。
复盘解读阶段允许大模型做语义分析。
```

日报本体保持规则化、可复现、轻量，不因模型接口、网络、费用或随机性影响生成结果。大模型只在解读阶段用于解释、归因、识别遗漏和辅助交易复核。

## 解读顺序

```text
市场环境 → 四灯信号 → 主线阶段 → 早期/确认信号 → 催化语义分析 → ETF/中军载体 → 操作框架 → 风险点
```

## 大模型语义分析做什么

- 识别标题/摘要中的语义主题，而不只看字符串命中；
- 将语义主题映射到可能相关的行业、概念和 ETF 载体；
- 判断文本催化是政策、产业、资金、公告、研报还是情绪拥挤；
- 识别价格强但催化弱、文本热但价格弱、退潮仍叙事热等背离；
- 发现日报规则层可能漏掉的潜在催化线索；
- 为用户解释交易信号质量、复核优先级和风险点。

## 大模型语义分析不做什么

- 不改写日报中的 A/B/C/退潮评级；
- 不回填历史日报指标；
- 不把文本热度单独升级为主线；
- 不让退潮行业因新闻热度直接恢复候选；
- 不绕过 ETF/中军载体、环境分、偏离度和回撤条件。

## 典型解读方式

```text
“AI 算力需求拉动”可语义映射到 CPO、光模块、服务器、液冷、PCB、半导体设备等方向。
若这些方向没有同步进入价格/宽度修复，只作为潜在线索；
若其中某些方向已出现早期信号、概念+行业共振且有 ETF/中军载体，则提高复核优先级。
```

最终输出应回答：

```text
能不能买 → 值不值得看 → 要不要躲 → 需要复核什么条件
```
