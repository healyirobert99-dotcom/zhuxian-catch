# 项目记忆

## ⚠️ 最高优先级：数据完整性
- 数据的真实性、时效性、完整性是系统基石，不可妥协
- 日报生成前必须验证当日 stock_daily / stock_daily_basic 行数 >=5000，主要指数数据完整
- 数据补齐过程中遇到任何 Tushare API 报错、限速、缺字段等问题，必须停下来寻求用户确认
- 禁止在数据不完整的情况下生成日报或给出解读
- 数据库文件损坏或锁冲突时先报告再处理，不能自行删除或重建

## 日报解读约定
- 用户要求日报出来后直接给交易建议，不要「不构成投资建议」之类的免责声明
- 关注三件事：信号变化、值不值得动（A 级/环境转绿时直接说方向+节奏）、风险在哪（过热不追/退潮避开）

## 日报解读会商机制
- 日报生成 → 用户发指令要求解读 → 助手独立输出解读（不混入日报）
- 解读优先级：能不能买 → 值不值得看 → 要不要躲
- 风格：3~5 条要点，给具体仓位/止损/加仓条件，不用术语（必有术语时加解释）
- 早期信号出现时给具体框架 + 条件判断
- 用户风格：β 收益导向，需要板块效应（宽度、中军、ETF）。没有板块效应的个股行情直接提醒不适合参与。
- 有 ETF 时附 ETF 代码，无 ETF 时查中军/弹性标的，都没有时明确标「无板块效应」
