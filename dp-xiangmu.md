## 6. 数据补齐经验（重要！避免重复踩坑）

### 6.1 stock_daily 补齐
- 使用 `pro.daily(trade_date=YYYYMMDD)` 逐日拉取
- Tushare 免费版约 3 调用/秒，大批量补齐时要等（1678 日期 ≈ 8-10 分钟）
- 批量写入 SQLite 时用 `executemany` + `pragma synchronous=off` 加速
- 关闭 WAL 模式后清理 `-wal` / `-shm` 文件避免锁冲突
- 不要同时开多个 Python 进程写同一个 SQLite！

### 6.2 stock_basic 补齐
- `pro.stock_basic(exchange='', list_status='')` 获取含退市股的全部股票
- 然后 upsert 到 stock_basic 表
- 历史回测需要退市股的行业信息做 JOIN

### 6.3 concept_daily 补齐（关键发现）
- ❌ `pro.concept_daily()` 在 Tushare 1.4.29 中不可用
- ✅ 正确 API: **`pro.dc_index(trade_date=dt, idx_type='概念板块')`**
- 覆盖: 2025-01-01 ~ 现在（逐日可用，18 个月历史）
- 每次返回 ~450-490 个概念板块
- 包含字段: ts_code, trade_date, name, pct_change, turnover_rate, up_num, down_num, total_mv, leading, leading_pct

### 6.4 concept_member 补齐
- 使用 `pro.dc_member` API
- 每日日报运行时会自动同步

## 8. 用户偏好与解读约定

### 8.1 解读会商机制
- 日报生成后，用户发送指令要求解读，助手独立输出（不混入日报正文）
- 解读按操作优先级排列：**能不能买 → 值不值得看 → 要不要躲**
- 每类附具体仓位、止损/加仓条件、ETF 代码或中军标的

### 8.2 解读风格
- 3~5 条要点，直白口语化，不用金融术语（必有术语时括号加解释）
- A 级或早期信号出现时：给具体数字框架 + 条件判断
- 有 ETF 的行业：附 ETF 代码
- 无 ETF 的行业：查中军/弹性标的 → 都没有则标「🚫 无板块效应，不适合 β 策略」
- 退潮标的：直接说跑还是减

### 8.3 用户交易风格
- **β 收益导向**：只看板块级别的行情，不参与个股炒作
- 关注板块宽度（站上均线的股票比例）和中军（大市值龙头）是否存在
- 没有板块效应的信号（宽度低、无中军、无 ETF）= 不适合参与，直接跳过
- 过热不追（涨停/偏离过大），退潮避开

## 9. 系统关键概念

### 9.1 主线级别
- **A 级**: 高优先级，强度/宽度/生命周期完整
- **B 级**: 可持续研究，可能有瑕疵
- **C 级**: 升温/修复/弱势反弹，需验证
- **退潮**: 明显转弱，主要做风险复核
- **低频监控**: 暂时不是核心方向

### 9.2 四灯信号
| 灯 | 含义 | 逻辑 |
|---|---|---|
| 🔴🟢 环境 | 市场评分 ≥55? | 代码: `lamp_env = "🟢" if env_score >= 55 else "🔴"` |
| 🔴🟢 方向 | 有优先复核行业? | 代码: `lamp_direction = "🟢" if not priority.empty else "🔴"` |
| 🔴🟢⚪ 共振 | 行业-概念同向? | 灰=无数据, 红=无共振, 绿=有共振 |
| 🔴🟢⚪ 时机 | 早期不偏热? | 灰=无时机数据, 红=有情绪顶点, 绿=未见顶点 |

### 9.3 环境分区间
| 分数 | 含义 |
|---|---|
| 0-29 | 防守/弱势 |
| 30-44 | 弱观察 |
| 45-54 | 偏观察 |
| 55-69 | 中性偏强（可操作门槛） |
| ≥70 | 进攻 |

### 9.4 早期主线优先复核
- 触发: `企稳重估 / 重新升温 / C级结构修复` + 环境分 >= 45
- 标签: `priority_label = "优先复核"`

### 9.5 概念+行业共振
- 概念成分股 → 统计最高频行业 → 作为对应行业
- 对应行业 A/B/C → ✅ 共振
- 对应行业 退潮/低频 → ❌ 背离

### 9.6 日报阶段用词映射
内部逻辑不变，日报展示时自动替换为易读标签：

| 内部用词 | 日报显示 | 内部用词 | 日报显示 |
|---|---|---|---|
| 强确认延续 | 强势确认 | 确认后退潮 | 逐步退潮 |
| 候选主线 | 待观察 | 退潮主线 | 退潮 |
| 弱势反弹 | 弱反弹 | 暂不观察 | 无信号 |
| 真延续 | 趋势延续 | 低频监控 | 低关注 |
| 高位震荡 | 高震 | 峰值回撤后修复 | 回撤修复 |
| 企稳重估 | 🛡 企稳 | 重新升温 | 🔥 回暖 |
| C级结构修复 | 🔧 修复 | 升温观察 | 预热中 |

### 9.7 催化复核与解读辅助层

- 目的：帮助助手在用户要求“系统解读日报”时判断信号质量，不做资讯完整性追求。
- 输入：`data/catalysts/catalyst_titles.csv`，字段为 `date,source_type,source_name,title,summary,related_industry,related_concept`
- 关键词：`config/catalyst_keywords.csv`
- 自动同步：`scripts/sync_catalysts.py`，手动严格生成入口会先尝试同步最近 5 天标题。
- 摘要层：`summary` 可空；有摘要时与标题一起匹配关键词和行业/概念。
- 正文输出：`催化复核与交易解读提示`，最多 3-5 条。
- 附录输出：`附录 F：催化关键词命中明细`。

解读顺序：

```text
环境灯 → 主线阶段 → 早期/确认信号 → 催化验证 → ETF/中军载体 → 操作框架 → 风险点
```

硬约束：

- 文本不能改变 A/B/C/退潮评级；
- 文本不能让退潮行业直接恢复候选；
- 文本不能单独触发建仓或加仓；
- 研报/情绪关键词密集时优先检查是否接近叙事高峰。

## 10. 交易策略范式（建仓→加仓→持有→清仓）

> 基于 2021-2026 年 2653 条早期信号回测数据逐条验证。适用于 β 策略（ETF/中军），不适用于弹性标的个股。

### 10.1 总体原则

| 原则 | 内容 | 数据支撑 |
|---|---|---|
| 载体 | 优先 ETF，其次中军。无 ETF 且无中军的信号直接跳过 | — |
| 持有期 | 40 个交易日，20 天太短、60 天边际提升仅 +2~3pp 但回撤恶化 3pp | 验证 3 |
| 总仓位上限 | 单信号 ≤15%，总仓位（跨信号）≤60% | — |
| 环境分 <45 | 暂停所有新增建仓，已有仓位按信号级别止损管理（不因环境清仓） | 验证 7：弱市企稳胜率 77.1% |

### 10.2 四阶段操作表

```
信号出现          第 1~3 天回踩确认     第 5~10 天峰值窗口      第 40 天到期
   ↓                  ↓                    ↓                      ↓
 Phase 1: 轻仓建仓    Phase 2: 确认加仓     Phase 3: 持有/分批减    Phase 4: 清仓
```

#### Phase 1 — 轻仓建仓（信号日+1~3 天）

| 信号类型 | 建仓比例 | 载体优先级 | 回撤止损 | 前置条件 |
|---|---|---|---|---|
| 🛡 企稳 | 8% ~ 10% | ETF → 中军 → 跳过 | 持仓回撤达 -7% | 环境分 ≥45 |
| 🔥 回暖 | 5% ~ 8% | ETF → 中军 → 跳过 | 持仓回撤达 -7% | 环境分 ≥45；≥70 时仓位减半 |
| 🔧 修复 | 5% | ETF → 中军 → 跳过 | 持仓回撤达 -7% | 环境分 ≥45，60 日回撤 >-15% |

**操作细节：**
- 信号后 56-61% 的样本在 5 日内下跌，等回踩再建可将胜率从 59-69% 提升到 65-75%（验证 1）
- 如果 3 天内继续大涨（偏离 MA20 > +8%），跳过——已过最佳窗口
- 建仓后设回撤止损 -7%（验证 5：回撤 <5% 胜率 88-93%，>7% 仅 50-59%）
- **不要用「信号日低点 -3%」止损**——82% 的样本都会触发，是无效止损

#### Phase 2 — 确认加仓（第 5~10 天，满足条件才加）

**宽度对胜率无区分力，不绑加仓条件**（验证 4：宽度 30% 和 70% 胜率一致）。

**企稳加仓条件（任一满足即可加 5%）：**
- 系统级别从企稳升级为 C 级观察以上
- 连续 5 天站上 MA20，且回撤未触及 -7%

**回暖加仓条件（任一满足即可加 5%）：**
- 连续 3 天站上 MA20
- 行业 5 日成交额持续放大
- **禁忌：环境分 ≥70 时不加仓，仓位降回 5%**（验证 7：进攻中回暖胜率仅 55.3%）

**修复加仓条件：**
- 系统阶段从修复升级为预热中或待观察

**累计仓位上限：** 企稳 ≤15%，回暖 ≤12%（进攻环境 ≤8%），修复 ≤10%

#### Phase 3 — 持有/分批减仓（第 5~20 天）

峰值中位数在 **第 8-9 天**，54% 的峰值在 10 天内出现（验证 2）。均值 13 天是被长尾拉高的。

**分批减仓触发（任一条）：**
- 达到预期峰值：企稳 +3.4%、回暖 +4.4%、修复 +3.7% → 减 1/3（验证 6：仅 33% 信号能达峰，但达峰后最终超额 +5-6%）
- 持仓回撤触及 -7% → 减 1/2
- 连续 3 天收盘价低于 MA20 → 减 1/2
- 系统发出退潮预警或降级 → 全部清仓

**持有观察：**
- 行业级别稳定或升级，回撤未触及 -7%，未到 40 天

#### Phase 4 — 清仓（满足任一条件）

| 清仓条件 | 操作 | 数据 |
|---|---|---|
| 持仓达 40 个交易日 | 全部清仓，不管盈亏 | 验证 3：40→60 胜率仅 +2-3pp，回撤恶化 3pp |
| 持仓回撤触及 -7% | 全部清仓 | 验证 5：回撤 >7% 后胜率 50-59% |
| 行业调入退潮或低关注 | 全部清仓 | 信号本身失效 |
| 环境分 <30 | **不清仓**，用信号级别止损管理 | 企稳在 <30 中胜率 70.6% |

### 10.3 多信号并发时的仓位管理

| 场景 | 策略 |
|---|---|
| 同一天多个企稳信号 | 选 1 个建仓（优先有 ETF 的），其他加入观察列表 |
| 企稳+回暖同时出现 | 企稳按 Phase 1 建，回暖只看不动等确认 |
| 已有持仓又出新信号 | 新信号按 Phase 1 建，不挪用已有仓位 |
| 总仓位已到 60% | 不新建仓，只做已有仓位的管理 |

### 10.4 环境分动态调节

| 环境分 | 调节动作 | 数据依据 |
|---|---|---|
| 0-29 | 不新建仓；已有仓位不因环境清仓，用信号回撤止损 | 企稳在 <30 中胜率 70.6%（34 样本） |
| 30-44 | 不新建仓；已有仓位可持有但不可加仓 | — |
| 45-54 | 企稳优先（胜率 77.1%），回暖减半，修复谨慎 | 验证 7 |
| 55-69 | 正常建仓；回暖需等 3 天回踩确认后再建 | — |
| ≥70 | 正常建仓；回暖仓位减半（胜率仅 55.3%），不加仓 | 验证 7 |

## 11. 回测验证速查

| 验证项 | 结论 |
|---|---|
| 1. 等 3 天建仓 | ✅ 56-61% 信号先跌后涨，等回踩可将胜率提 6-8pp |
| 2. 峰值窗口 | 中位 8-9 天（非均值 13 天），54% 在 10 天内 |
| 3. 40 天清仓 | ✅ 40→60 胜率仅 +2-3pp，但回撤恶化 3pp |
| 4. 宽度加仓 | ❌ 宽度对胜率无区分力，30% 和 70% 一致 |
| 5. 回撤止损 -7% | ✅ 回撤 <5% 胜率 88-93%，>7% 仅 50-59% |
| 6. 达峰减仓 | ✅ 33% 能达峰，达峰后最终超额 +5-6%，未达峰仅 +0.3-1.6% |
| 7. 环境分与胜率 | 企稳偏观察 77.1% > 进攻 68.7%；回暖偏观察 63.6% > 进攻 55.3% |

| 问题 | 影响 | 状态 |
